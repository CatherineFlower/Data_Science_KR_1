# Работа с ветками в Git

## Создание новой ветки

### Вариант 1: Создать и переключиться сразу
```bash
git checkout -b имя-ветки
```

### Вариант 2: Новый синтаксис (Git 2.23+)
```bash
git switch -c имя-ветки
```

### Вариант 3: Создать ветку без переключения
```bash
git branch имя-ветки
git checkout имя-ветки
```

## Просмотр веток

```bash
# Все локальные ветки
git branch

# Все ветки (включая удалённые)
git branch -a

# Текущая ветка (показывает *)
git branch
```

## Переключение между ветками

```bash
git checkout имя-ветки
# или
git switch имя-ветки
```

## Удаление ветки

```bash
# Удалить локальную ветку (после переключения на другую)
git branch -d имя-ветки

# Принудительное удаление
git branch -D имя-ветки

# Удалить удалённую ветку
git push origin --delete имя-ветки
```

## Отправка ветки на удалённый репозиторий

```bash
# Первая отправка (с установкой связи)
git push -u origin имя-ветки

# Последующие отправки
git push
```

## Переименование ветки

```bash
# Переименовать текущую ветку
git branch -m новое-имя

# Переименовать другую ветку
git branch -m старое-имя новое-имя
```

## Примеры имён веток

- `feature/название-функции` - для новой функциональности
- `bugfix/описание-бага` - для исправления ошибок
- `hotfix/срочное-исправление` - для срочных исправлений
- `develop` - для разработки
- `release/версия` - для релизов

## Полезные команды

```bash
# Посмотреть последний коммит в каждой ветке
git branch -v

# Посмотреть, какие ветки уже слиты
git branch --merged

# Посмотреть, какие ветки не слиты
git branch --no-merged
```
